Conceptos de desarrollo de software basado en componentes (DSBC)

Regresar

es una forma de crear y gestionar sistemas de software a partir de componentes de software reutilizables, modulares e independientes, que se comunican entre sí mediante interfaces bien definidas. Este enfoque busca mejorar la calidad, la productividad, la flexibilidad y la fiabilidad del software, al reducir la complejidad, el costo y el tiempo de desarrollo. Algunos conceptos clave del DSBC son

Componente de software: es una unidad de software que ofrece un conjunto de servicios a través de sus interfaces, y que puede ser usado e integrado con otros componentes sin necesidad de conocer su implementación interna. Un componente puede ser tan simple como una función o tan complejo como un subsistema. Un componente puede estar escrito en cualquier lenguaje de programación y puede ser distribuido o local. Interfaz de componente: es la especificación de los servicios que ofrece un componente, así como de los requisitos que necesita para funcionar. La interfaz define el nombre, los parámetros, el tipo de retorno y la descripción de cada servicio. La interfaz también puede incluir eventos, propiedades y atributos del componente. La interfaz es la única forma de acceder al componente desde el exterior, y garantiza el encapsulamiento y la abstracción del componente. Arquitectura de software: es la estructura o el diseño de alto nivel de un sistema de software, que describe los componentes que lo conforman, las relaciones entre ellos, y los principios y patrones que guían su organización. La arquitectura de software facilita la comprensión, el análisis, el diseño, la implementación, el mantenimiento y la evolución del sistema. Marco de trabajo: es un conjunto de componentes, herramientas, estándares y metodologías que proporcionan una base común para el desarrollo de aplicaciones de software. Un marco de trabajo define la arquitectura, las interfaces, los servicios y las funcionalidades básicas de un dominio o una tecnología específica. Un marco de trabajo permite la reutilización, la integración y la adaptación de los componentes que lo conforman o que se desarrollan sobre él. Modelo y plataforma de componentes: es un conjunto de conceptos, reglas, especificaciones y tecnologías que definen cómo se crean, se describen, se almacenan, se buscan, se distribuyen, se instalan, se configuran, se ejecutan, se comunican y se gestionan los componentes de software. Un modelo y una plataforma de componentes proporcionan los mecanismos necesarios para tratar la complejidad y la heterogeneidad de los sistemas abiertos y distribuidos. Algunos ejemplos de modelos y plataformas de componentes son COM, CORBA, JavaBeans, EJB, .NET, etc.
